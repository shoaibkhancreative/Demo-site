# NLT Exclusive Mentorship — Website

A marketing site + frictionless checkout for the NLT trading mentorship (7 chapters, 99 classes). No sign-up, no backend — payment is crypto-only and orders hand off to your existing Telegram bot (`@exclusive_mentorship_bot`).

## Tech stack

- **React 19 + Vite** — plain JavaScript (JSX), no TypeScript
- **React Router v7** — client-side routing (Home, 7 chapter pages, Checkout)
- **Tailwind CSS v4** — design tokens live in `src/index.css` (`@theme` block)
- **lucide-react** — icons

Static output only — `npm run build` produces a `dist/` folder you can host anywhere (Vercel, Netlify, S3+CloudFront, your own Nginx box, etc). There is no server/API in this project.

## Getting started

\`\`\`bash
npm install
npm run dev       # local dev server

## How the Telegram handoff works

Step 5's "Confirm & Join" → "Send Proof of Payment" button opens:

\`\`\`
https://t.me/exclusive_mentorship_bot?text=<url-encoded order summary>
\`\`\`

**This is deliberately \`?text=\`, not \`?start=\`.** Telegram's bot deep-link \`start\` parameter is capped at 64 characters and only allows \`A-Z a-z 0-9 _ -\` — your own bot's \`parseOrderPayload()\` reflects exactly that limit, which is why it's a terse \`Plan-ChapterCount-Addons-Total\` code with no room for a name, DOB, phone, or email. \`?text=\` has no such limit: it opens the chat with a full draft message that the user reviews and taps **Send** on (that manual tap is a deliberate Telegram anti-spam measure, not a bug).

Once sent, the message lands as a normal private text message — which your bot already routes straight into the correct admin support thread via \`handleUserTextMessage()\` → \`routeMessageToSupportThread()\`. **No changes to your bot are required for this to work.**

The message format (see \`src/lib/telegram.js\` → \`buildOrderMessage\`) is plain, emoji-labeled text designed to be read by a human admin, e.g.:

\`\`\`
🎓 New Enrollment — NLT Mentorship

📦 Plan: Full Bundle (All 7 Chapters)

💰 Total: \$79 USDT
──────────────
👤 Name: ...
🎂 DOB: ...
📧 Email: ...
──────────────
💳 Payment Method: USDT (TRC20)
✅ Terms & Trading Risk: Acknowledged
\`\`\`

If you later want the bot to parse this automatically into a structured order (rather than a human reading it), the consistent emoji/label-per-line format is easy to regex against.

## Two business rules mirrored from your bot

While reading \`telegram-crm-bot.js\` to design the handoff, two things stood out that this site intentionally mirrors — flagging them explicitly in case either was accidental rather than by design:

1. **Chapters unlock sequentially.** Your bot's cart (\`getNextAvailableChapter\`) only ever offers the *next* chapter, never an arbitrary one — so the site's "Individual Chapters" picker in Step 1 works the same way (pick chapter *N*, get 1..*N*). If you actually want à la carte chapter picking, that needs a bot-side change too, since the DB schema (\`orders.chapters\`) could technically support it even though nothing currently produces that.
2. **Your bot's \`PAYMENT_INSTRUCTIONS_TEXT\` includes a "Bybit UID" option** that wasn't in the original site spec (which listed 5 methods: Binance UID, USDT TRC20/BEP20, BTC, ETH). The site currently matches the spec's 5 methods — add Bybit to \`PAYMENT_METHODS\` in \`constants.js\` if you want the two to match exactly.

## Deployment (SPA routing)

This is a single-page app using client-side routing, so your host needs to serve \`index.html\` for unknown paths (otherwise refreshing \`/checkout\` or sharing a direct chapter link 404s). Already included:

- \`vercel.json\` — Vercel rewrite rule
- \`public/_redirects\` — Netlify redirect rule

For Nginx/Apache/other hosts, add an equivalent "fall back to index.html" rule (\`try_files \$uri /index.html;\` on Nginx).

## Design system

Tokens are in \`src/index.css\` under \`@theme\`. Quick reference:

| Token | Use |
|---|---|
| \`--color-canvas\` | Page background (bright cool off-white) |
| \`--color-ink\` | Primary text / dark surfaces |
| \`--color-gold\` | Primary accent — CTAs, price, "Smart Money" motif |
| \`--color-signal\` | Secondary accent — links, info states |
| \`--color-bull\` / \`--color-bear\` | Used only inside the market-structure illustration |

Fonts: **Space Grotesk** for display/headlines (Latin only — Bengali falls back to Hind Siliguri automatically within the same heading), **Hind Siliguri** for body copy (Bengali + Latin). Both loaded via Google Fonts in \`index.html\`.
